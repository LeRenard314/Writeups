К директории `.git` может свободно обращаться любой желающий.
Это опасно, т.к прочитав содержимое директории, атакующий может найти много информации про исходный код, логику запросов, захардкоженные креды и т.д.
Попробуем стянуть найденную директорию с помощью утилиты `git-dumper`:

```Shell
python3 git_dumper.py https://<labID>.web-security-academy.net/.git ./dumped
```

Ну или прибегнув к запросу через `wget`:

``` Shell
wget -r https://<labID>.web-security-academy.net/.git
```

В итоге стянется следующее:

![placeholder](images/labDumpedContent.png)

Пробуем чекнуть содержимое `.conf` файла, но к сожалению, захардкоженного пароля там не нашлось:

![placeholder](images/labAdminEnv.png)

Тогда пробуем посмотреть отличия файлов в истории коммитов:

``` Shell
git log
```

Видим, что всего сделано два коммита:

![placeholder](images/labGitLog.png)

Попробуем просмотреть информацию про первый коммит:

``` Shell
git show <commitHash>
```

Наблюдаем, что в сатрой версии пароль администратора был прописан в коде, который остался в истории коммитов, несмотря на то что код в последующих версиях притерпевал изменения:

![placeholder](images/labNewPwd.png)

В решении от Portswigger намекали на решение при помощи `git diff`. Эта команда сравнит и покажет отличия текущего коммита (`HEAD`) от предыдущего  коммита до `HEAD`(`HEAD~1`). В контексте данной лабы это актуально, поскольку коммитов всего два, и свинья порылась именно в первом:

``` Shell
git diff HEAD~1
```

![placeholder](images/labDiffHead.png)

Теперь воспользовавшись извелеченной информацией можно залогиниться от имени пользователя `administrator`, перейти на панель управления и избавиться от пользователя `carlos`:

![placeholder](images/labSolved.png)

## Противодействие

Нельзя допускать попадания служебной директории `/.git` в продакшн среду. Пароли не прописывать в коде, а указаывать в `.env` файлах проекта, убедиться что `.gitignore` прописан верно, не допуская коммит чувствительных данных. При CI/CD использовать инструменты по обнаружению сектретов: `truffleHog`, `git-secrets` и др. Скомпромитированные данные подлежат незамедлительной замене.
